# 00 — 요건 정리

브레인스토밍을 통해 결정된 요건. 이후 모든 디자인 섹션의 기반.

## 목적

사용자가 Threads에서 **본인이 리포스트한 게시물**을 Obsidian vault에 저장하기 위한 Chrome 확장프로그램.

## 사용 방식 (Trigger)

- **수동, 1건씩** 저장
- 트리거 진입점:
  - 확장 아이콘 클릭
  - 단축키 `Ctrl+Shift+S` (변경 가능)
- **개별 게시물 페이지**(`threads.com/@user/post/<id>`)에서만 동작. 피드/스크롤 페이지에서는 비활성화.

## 저장 채널

- **Obsidian Local REST API 플러그인** 사용
- 백그라운드 저장 — Obsidian이 자동으로 열리지 않음
- 사용자가 옵션 페이지에서 API URL과 토큰을 입력

## 캡처할 데이터

한 게시물 = "**원 작성자가 자기 자신의 댓글로 본문을 이어 쓴 모든 글의 묶음 + 모든 이미지**".

- 원 게시물 본문 + 작성자가 이어쓴 댓글들 (다른 사람 댓글은 **제외**)
- 모든 이미지 (Obsidian vault에 다운로드)
- 작성자 핸들 (`@username`)
- 게시일, 저장일
- 원본 URL

## 파일 구조

vault 안의 고정 폴더 `Thread/` 아래, **노트 1개당 폴더 1개**:

```
Thread/
├── 2026-05-05 @username 첫줄을최대30자까지자른요약/
│   ├── note.md          ← 본문 (이미지를 ./img1.jpg 같은 상대경로로 참조)
│   ├── img1.jpg
│   └── img2.jpg
└── ...
```

폴더명 규칙:
- 형식: `{게시일 YYYY-MM-DD} {작성자 핸들} {첫줄 최대 30자}`
- 게시일은 게시물의 작성일 (저장일 아님)
- Windows 금지문자(`\ / : * ? " < > |`) 자동 제거
- 폴더명 우연 충돌 시 ` (2)`, ` (3)` 자동 suffix

## 마크다운 형식 (`note.md`)

```markdown
---
source: https://threads.com/@user/post/abc123
author: "@username"
posted_at: 2026-05-04T18:23:00Z
saved_at: 2026-05-05T10:30:00Z
tags: [threads]
---

# {첫 게시물 첫 줄}

{첫 게시물 본문}

![[img1.jpg]]

---

{이어쓴 댓글 1}

![[img2.jpg]]

---

{이어쓴 댓글 2}
```

## 중복 처리

같은 게시물 URL을 다시 저장 시도 시 → **스킵 + 알림**: "이미 저장됨" + 기존 노트 열기 버튼.

## 설정 (Options 페이지)

| 항목 | 기본값 | 비고 |
|---|---|---|
| API 호스트 | `https://127.0.0.1:27124` | Local REST API 플러그인 주소 |
| API 토큰 | (비어있음) | 사용자가 플러그인에서 발급받아 입력. 필수. |
| 저장 폴더 | `Thread` | vault 내 경로 |
| 단축키 | `Ctrl+Shift+S` | `chrome://extensions/shortcuts`에서 변경 |

## 알림·에러 처리

- **저장 성공** → Chrome 알림: "Obsidian에 저장됨" + 클릭 시 노트 열기
- **저장 실패** → Chrome 알림 + 에러 종류 표시
- 에러 케이스:
  - Obsidian 미실행 또는 API 미응답 → "Obsidian을 켜주세요"
  - API 토큰 누락/잘못됨 → "설정에서 API 토큰을 확인하세요"
  - DOM 추출 실패 → "페이지 새로고침 후 다시 시도"
  - 이미지 일부 다운로드 실패 → 본문은 저장하고 노트 끝에 `<!-- 이미지 N개 누락 -->` 코멘트 추가
  - 중복 → "이미 저장됨, 기존 노트 열기"

## 기술 접근법

**DOM 스크래핑** (content script 주입). 이유:
- 비공식 GraphQL API보다 깨지더라도 selector만 고치면 됨 (예측 가능한 유지보수)
- oEmbed API는 이어쓴 댓글 스레드를 못 줘서 핵심 기능 불가
